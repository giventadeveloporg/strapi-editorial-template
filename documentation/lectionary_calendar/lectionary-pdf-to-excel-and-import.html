Lectionary PDF → Excel & Strapi Import

This document describes the input PDFs for the liturgical calendar, how their content maps to Excel columns and Strapi fields, and the scripts used to delete, import, and export liturgy-day data. The PDF-based import replaces the earlier seed script for bulk data.

1. Input PDFs (which is which)

📄 Source files

Two PDFs in documentation/lectionary_calendar/ are the input data. The parser and import scripts use them as follows.

FileLanguageUse
2026-Liturgical-Calender.pdf English Day headings (e.g. “New Year. Circumcision of our Lord…”), season codes (Niram N). One entry per calendar day in format N. Description.
Panjangom_26.pdf Malayalam Same structure: day number + description. Season codes appear as (\ndw N). Merged with English by date.

Entries are merged by date (YYYY-MM-DD). Month is inferred from the order of days (day 1 starts a new month). If a date exists only in one PDF, that row keeps the other language fields empty until filled by carry-over or backfill where applicable.

2. PDF → Excel / Strapi column mapping

⚙️ Column mapping

Each parsed liturgy day becomes one row in Excel and one Strapi document. English and Malayalam columns are on the same row (not repeated rows per language).

Excel / Strapi fieldSourceNotes
dateParsed from both PDFsYYYY-MM-DD; inferred from day number and month order.
dayHeadingEnEnglish PDFText after N. for that date (continuation lines merged).
dayHeadingMalylmMalayalam PDFText after day number for same date (merged by date).
seasonNameEnExtracted from EN heading(Niram N) → “Niram N”; else carry-over from previous day or backfill.
seasonNameMalylmExtracted from ML heading(\ndw N) → “Niram N”; else carry-over / backfill.
orderImport script0-based index of the day in the created set.
readings—PDFs do not list readings; array is empty. Excel columns reading1_*, reading2_* are for manual or future data (e.g. liturgyHeadingEn, liturgyHeadingMalylm, contentPlaceEn, contentPlaceMalylm).

Excel template columns for readings (when populated from another source): reading1_liturgyHeadingEn, reading1_liturgyHeadingMalylm, reading1_contentPlaceEn, reading1_contentPlaceMalylm, and similarly reading2_*. These are not filled by the PDF parser.

3. Delete liturgy days (purge)

📋 Delete script

Use delete-liturgy-days.js to remove liturgy-day entries before re-importing. With no tenant ID, it deletes all liturgy days (all tenants). With --tenant-id=..., it deletes only that tenant’s entries.

Purge liturgy days
🔧 Customize parameters
node scripts/delete-liturgy-days.js --tenant-id=tenant_demo_002
Windows single-line:
node scripts/delete-liturgy-days.js --tenant-id=tenant_demo_002
Note: Omit --tenant-id to delete all liturgy days across tenants. Use DRY_RUN=1 to only log the count without deleting.

4. Import from PDF into Strapi

📋 Import script

Use import-liturgy-days-from-pdf.js to parse the two PDFs and create liturgy-day documents in Strapi for a tenant.

Import from PDF
🔧 Customize parameters
node scripts/import-liturgy-days-from-pdf.js --tenant-id=tenant_demo_002 --year=2026 --limit=10
Windows single-line:
node scripts/import-liturgy-days-from-pdf.js --tenant-id=tenant_demo_002 --year=2026 --limit=10

Options:

  • --tenant-id=... — Tenant to attach entries to (default: tenant_demo_002).
  • --year=YYYY — Calendar year for parsed dates (default: 2026).
  • --limit=N — Create at most N entries (for testing).
  • DRY_RUN=1 — Parse and log only; do not create.
  • --dump-text — Write raw PDF text to dumped-en.txt / dumped-ml.txt.
  • --en-pdf=path / --ml-pdf=path — Override PDF paths.

5. Export parsed PDF to Excel

📊 Export to Excel

To get the same parsed data into an Excel sheet (e.g. for review or editing), use export-liturgy-days-from-pdf-to-excel.js. It runs the same parser as the import script and writes all merged days to documentation/lectionary_calendar/liturgy-days-from-pdf.xlsx with columns aligned to the mapping above.

Export PDF → Excel
🔧 Customize parameters
node scripts/export-liturgy-days-from-pdf-to-excel.js --year=2026
Windows single-line:
node scripts/export-liturgy-days-from-pdf-to-excel.js --year=2026

Output file: documentation/lectionary_calendar/liturgy-days-from-pdf.xlsx. Reading columns are present but empty (PDFs do not contain per-day readings).

6. Run order

  1. Stop the Strapi server if it is running.
  2. Delete liturgy days for the target tenant (or all) so re-import does not duplicate.
  3. Import from PDF (optionally with --limit=10 or DRY_RUN=1 first).
  4. Optionally run Export to Excel to inspect or edit the parsed data in a spreadsheet.
  5. Start Strapi and verify in Content Manager → Liturgical Calendar – Day.

7. Scripts and pipeline

🔗 Implementation
ScriptRole
lectionary/pdf-extract.jsUses pdf-parse to extract raw text from a PDF; used by --dump-text and by the parser.
lectionary-pdf-parser.jsParses EN/ML PDF text into day entries (date, headings, season), merges by date, extracts (Niram N) / (\ndw N), carry-over and backfill for season.
import-liturgy-days-from-pdf.jsLoads Strapi, calls parser, creates liturgy-day documents; supports --dump-text, --limit, DRY_RUN.
delete-liturgy-days.jsDeletes liturgy-day entries (all tenants or one tenant).
export-liturgy-days-from-pdf-to-excel.jsRuns parser and writes rows to liturgy-days-from-pdf.xlsx.

The PDF-based flow replaces the manual seed script (seed-liturgy-days.js) for bulk data. For a small fixed set of days, the seed script is still available; for full calendar data, use delete + import from PDF.
